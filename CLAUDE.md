# Summit Builder

Monorepo-free Laravel project.  
**Stack:** Laravel 11 · PHP 8.3 · Inertia.js v2 · React 19 · Tailwind v4 · FilamentPHP v3  
**Package manager:** Composer (PHP) + pnpm (JS assets)  
**Node:** 22 LTS · **PHP:** 8.3

---

## What this project is

A **multi-domain summit/funnel builder** — think WordPress but purpose-built for online summits and marketing funnels.

- Operators manage **Domains → Funnels → Steps** via a Filament admin panel
- Content editors add speakers, products, and page copy via Filament
- Visitors land on funnel pages rendered by Inertia + React (SSR-capable)
- Stripe handles checkout; Brevo handles email; ActiveCampaign handles CRM

**V1 scope:** Multi-domain funnel management + Filament CMS  
**V2 scope:** AI funnel generation + automation agents (Anthropic Claude)

## Key architectural decisions

### Multi-domain routing
Funnels are accessed via custom domains (e.g. `summit.clientsite.com`).

Flow:
1. `ResolveFunnelDomain` middleware runs on every request
2. Looks up `Domain` model by `request()->getHost()`
3. Binds the resolved `Funnel` into the request — controllers don't touch domains directly
4. Routes are defined with `Route::domain('{domain}')` or via middleware-based resolution

### Filament for CMS
Filament runs at `/admin` (or configurable path). Two panels:
- **Operator panel** — manage domains, funnels, steps, products
- **Editor panel** (optional, separate Filament panel) — content-only access for copywriters/designers

### Inertia for funnel pages
Controllers return `Inertia::render('Funnel/Optin', [...props])`.  
React handles rendering. No separate API — data flows through props.  
Blade is used **only** for `app.blade.php` (the Inertia shell).

### Stripe via Laravel Cashier
- `Order` model tracks completed purchases
- Stripe webhook listener updates order status
- Upsell steps use Stripe Payment Intents with customer already on file

### Queue / jobs
- Driver: Redis (Upstash)
- Jobs: `SendConfirmationEmail`, `SyncToActiveCampaign`, `ProcessStripeWebhook`
- Run locally: `php artisan queue:work`

## Deployment (Ploi + Hetzner)

- **Server:** Hetzner VPS (Ubuntu 22.04), managed via Ploi
- **PHP:** 8.3 via Ploi's one-click PHP provisioning
- **Database:** PostgreSQL — on-VPS for staging, Hetzner Managed DB for production
- **Process:** PHP-FPM + Nginx (Ploi configures this automatically)
- **Queue worker:** Supervisor via Ploi daemon
- **Deploy script (Ploi):**
  ```bash
  composer install --no-dev --optimize-autoloader
  php artisan migrate --force
  php artisan config:cache
  php artisan route:cache
  php artisan view:cache
  php artisan filament:cache-components
  pnpm install && pnpm build
  php artisan queue:restart
  ```
